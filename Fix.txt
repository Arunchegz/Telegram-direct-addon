**Bugs / correctness**

1. **`httpx.AsyncClient` in `_bot_channel_listener` lives outside the `while True` loop** — if a network error triggers the `except` branch and `asyncio.sleep(10)`, the context manager is never re-entered. Next iteration uses a closed client → errors. Move `async with httpx.AsyncClient` inside the loop or use a long-lived client like `state.py` does.

2. **`deferred_notifications` grows unbounded** — it's a plain dict, items only removed in `_prefetch_worker` via `.pop`. If a movie is evicted mid-prefetch without going through the worker's finally path, its `msg_id` leaks forever.

3. **`_hydrate_if_cached` accesses `download_manager._maps` directly** — breaks encapsulation and bypasses the `_lock`. Could race with `get_or_create` in a concurrent request. Use `download_manager.get_map()` + add an explicit hydrate method to `DownloadManager`.

4. **`/stream` for `tt`-prefixed series IDs never starts a download** — the movie/series routes call `_schedule(_ensure_download(...))`, but the `tt:` branch just returns streams with no download trigger. First play of an IMDB-keyed series goes straight to live proxy with no background caching started.

5. **`_progress_reporter` polls every 1s with a tight `asyncio.sleep(1)` loop** — if the movie is evicted mid-progress, `download_manager.get(movie_id)` returns `None` and the loop exits, but if task finishes and restarts (requeue), a second reporter task can spawn alongside an orphaned first one. `reporter.cancel()` in `_prefetch_worker` handles the happy path, but exceptions between `task._task` creation and `reporter.cancel()` can leak.

6. **`metrics.downloads_active` mutated directly in `downloader.py`** (`metrics.downloads_active += 1`) — bypasses the `threading.Lock()` in `Metrics`. Either use `await metrics.record_download_start()` or drop the lock since this is single-threaded asyncio.

7. **`import httpx` mid-file** — `main.py` has `import httpx` buried after `_sync_loop()`. Move to top with other imports.

8. **`except: continue` in `_sync_channel`** — bare `except` swallows `CancelledError`, blocking clean task shutdown during `lifespan` teardown. Use `except Exception: continue`.

---

**Reliability / edge cases**

9. **`_notify_edit` chat_id parse block duplicated** exactly in `_notify_send` and `_notify_edit`. Extract to `_resolve_chat_id(raw: str) -> int | str`.

10. **`_bot_channel_listener` and Pyrogram `_instant_sync_handler` both trigger `_sync_channel(force=True)` on new posts** — double sync every post when both `BOT_TOKEN` and `CHANNEL_USERNAME` are set. The `_sync_lock` prevents double-execution but wastes a redundant lock acquire. Consider `DISABLE_BOT_LISTENER=true` auto-set when Pyrogram listener is active.

11. **`_short_id` sha1 hash collision risk** — 10 hex chars = 40 bits. With hundreds of movies this is fine, but `del:` callback has no fallback if two movies collide. At minimum, log a warning if collision detected in `_handle_admin_callback`.

12. **`SparseFile.pread/_pwrite` open file on every call** — for high-throughput streaming (Path A), repeated `open(path, "rb")` per `LOCAL_READ_CHUNK` is expensive. Cache a `fileobj` per `SparseFile` with a lock, or use `os.pread`/`os.pwrite` with a persistent fd.

13. **`yield_file` has duplicate FloodWait/FileReferenceExpired handling** — the first-chunk block and the multi-chunk loop are nearly identical. Extract to a helper `_invoke_get_file(session, loc, off, chunk, c_idx, ...)` to reduce ~100 lines of duplication and divergence risk.

14. **`_session_lock` is global across all clients** — serializes media session creation for all DCs across all clients. Should be per-client (dict keyed by client id or index).

---

**Minor / style**

15. `_log_task_exception` print is too terse: `[task] {e}` loses type info. Use `print(f"[task] {type(e).__name__}: {e}")`.

16. `tg: Client = None` global used alongside `get_tg()` — `tg` is always set to `client_pool.primary()` at startup, making the global redundant. Can remove `tg` global and just use `get_tg()` everywhere.

17. `MANIFEST["idPrefixes"]` includes `"tt"` but the addon handles `tt`-prefixed IDs via Cinemeta passthrough, not as owned IDs. This causes Stremio to route all IMDB lookups here, adding latency even for movies not in TGStream. Consider removing `"tt"` and letting Cinemeta handle it natively unless you specifically want to intercept IMDB IDs.
